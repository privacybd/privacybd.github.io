SQL Injection Lab

본 사이트는 SQL Injection 공격의 원리와 위험성을 배우기 위한 시뮬레이터입니다.
입력값에 따라 동적으로 변하는 쿼리와 DB 반응을 실시간으로 확인해 보세요.

① Login Bypass (인증 우회)

비밀번호를 몰라도 '참(True)'을 만드는 구문을 삽입하여 관리자 계정으로 접속합니다.

SELECT * FROM users WHERE id='' AND pw='';
결과가 여기에 표시됩니다.
해설: 입력값에 ' OR '1'='1을 넣으면 SQL 쿼리 조건문이 항상 참이 됩니다. 엔진은 WHERE id='' OR '1'='1'로 해석하여 비밀번호 검증을 무시하고 첫 번째 계정(보통 관리자)을 반환합니다.

② UNION-based Injection (데이터 탈취)

상품 검색 기능을 악용하여 전혀 다른 테이블(사용자 정보)의 데이터를 불러옵니다.

SELECT name, price FROM products WHERE name='';
상품 목록이 여기에 표시됩니다.
해설: UNION 연산자는 두 쿼리의 결과를 합칩니다. 공격자는 기존 쿼리 뒤에 새로운 SELECT를 붙여 DB 계정 정보 등을 출력 화면에 포함시킵니다. 마지막의 --는 뒤의 쿼리 내용을 주석 처리하여 에러를 방지합니다.

③ Prepared Statement (방어 적용)

파라미터 바인딩을 적용하면 동일한 공격 코드가 단순한 '문자열'로 취급되어 작동하지 않습니다.

PREPARE stmt FROM 'SELECT * FROM users WHERE id=?';
EXECUTE stmt USING ;
결과가 여기에 표시됩니다.
해설: Prepared Statement는 쿼리 구조를 미리 컴파일하고 사용자 입력값은 나중에 따로 대입합니다. 따라서 입력값에 SQL 예약어(OR, UNION 등)가 있어도 구조가 변하지 않아 공격이 불가능합니다.
